# Dating Telegram Bot

## Этап 1 — Планирование и проектирование

- Описание сервисов
- Архитектура системы
- Схема базы данных
- Метрики и логирование (structlog + prometheus-client)

## Этап 2 — Разработка базовой функциональности 

- Подняты основные сервисы: `bot_service`, `profile_service`, `PostgreSQL`, `Redis`, `RabbitMQ`, `MinIO`
- Реализованы регистрация пользователя, заполнение/обновление анкеты и просмотр своей анкеты
- Подготовлен запуск через Docker Compose

## Этап 3 — Система анкет и ранжирования 

- Добавлен поиск анкет с лайками/пропусками и сохранением взаимодействий
- Реализована система ранжирования (первичный и поведенческий рейтинг)
- Добавлены системное меню команд Telegram и навигация кнопками в боте
- Реализованы ручной ввод имени, раздел/команда «Мои мэтчи» и выдача мэтчей из API

## Этап 4 — Дополнительные функции

- Настроены фоновые задачи через Celery + RabbitMQ:
  - периодический пересчет рейтингов (`recalculate_all_ratings`)
  - прогрев Redis-кэша выдачи анкет (`warm_discovery_cache`)
- Реализована интеграция с MinIO для фотографий:
  - `bot_service` отправляет фото анкеты в `profile_service`
  - `profile_service` загружает объект в MinIO и сохраняет URL в `profiles.photo_urls`
- Выполнена оптимизация производительности:
  - поведенческий рейтинг считается одним SQL-запросом вместо трех
  - добавлены индексы для критичных запросов discovery/matches/rating
  - добавлены метрики размера Redis-кэша и времени пересчета рейтинга
- Добавлены тесты для scoring-логики (`pytest`)
- Добавлен CI/CD: GitHub Actions workflow для автозапуска тестов profile_service
